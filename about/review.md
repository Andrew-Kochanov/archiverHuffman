### 1. Цель работы

В рамках учебной практики предстоит реализовать консольное приложение-архиватор на базе алгоритма Хаффмана, способное выполнять сжатие и восстановление файлов, а измерить степень сжатия и скорость работы получившегося приложения на данных разных типов и размеров. 

### 2. Теория

Алгоритм Хаффмана — один из наиболее известных алгоритмов сжатия данных без потерь, предложенный Дэвидом Хаффманом в 1952 году.  Алгоритм основан на понятии "префиксных кодов" — битовых последовательностей, обладающих тем свойством, что код каждого символа не является префиксом кода никакого другого символа. Это позволяет при прочтении битовой последовательности однозначно восстанавливать символы по их битовым кодам

**Принцип работы**:

1. Подсчёт частот — определяется частота встречаемости каждого символа во входных данных.
2. Построение дерева Хаффмана — каждый символ становится листом - узлом с весом, равным его частоте. Затем циклически выполняются следующие действия: из множества узлов выбираются два с наименьшими весами, создаётся новый внутренний узел с весом, равным сумме весов выбранных узлов, который становится их родителем. Процесс повторяется до тех пор, пока не останется один корневой узел.
3. Генерация кодов — выполняется обход дерева от корня к листьям. При переходе на левое поддерево к коду дописывается 0, при переходе на правое — 1. Кодом символа является битовая последовательность, полученная на пути от корня до соответствующего листа.

**Свойство оптимальности:**

Алгоритм Хаффмана гарантирует построение кода с минимально возможной средней длиной для заданного распределения частот при условии, что каждый символ кодируется отдельным кодовым словом.

### 3. Примерная функциональность

Программа должна поддерживать два режима через интерфейс командной строки:

- **Режим сжатия**:
    - Подсчёт частот всех байтов входного файла.
    - Построение дерева Хаффмана с использованием приоритетной очереди (реализованной например, через бинарную кучу).
    - Генерация таблицы кодов для каждого символа.
    - Запись заголовка (размер, количество символов, пары «символ-частота») в выходной файл.
    - Кодирование исходных данных: битовый буфер, запись байтов по мере заполнения.
- **Режим распаковки**:
    - Чтение заголовка, восстановление массива частот.
    - Построение дерева по частотам (аналогично этапу сжатия).
    - Побитовое чтение данных из файла и обход дерева до листа, вывод символа в выходной файл.
    - Проверка, что размер восстановленного файла совпадает с сохранённым в заголовке.

### 4. Технические реализации алгоритма:

- **SmnTin/huffman-archiver** – реализация на C с каноническими кодами Хаффмана, оптимизированная для больших файлов; демонстрирует высокую пропускную способность.
- **MichaelDipperstein/huffman** – простая ANSI C реализация с классическими и каноническими кодами, хорошо подходит для учебных целей.

### 5. Ожидаемые результаты исследования

- Для текстовых файлов (естественный язык, исходные коды программ) сжатие на 40–55%
- Для уже сжатых данных (JPEG, ZIP, PDF) – алгоритм неэффективен.
- Для очень маленьких файлов (< 1–2 КБ) сжатие может увеличивать размер из-за оверхеда заголовка и необходимости выравнивания битового потока до байта.
- Скорость распаковки обычно выше скорости сжатия, поскольку распаковка не требует построения дерева и таблицы.

### 6. Критерии успешной работы

- Программа корректно выполняет полный цикл «сжатие → распаковка» для файлов любых типов, что подтверждается сравнением исходного и восстановленного файлов (командой diff или побайтовым сравнением).
- Реализована обработка краевых случаев: пустые файлы, файлы с одним уникальным символом, файлы с большим количеством повторяющихся данных.
- Получены количественные оценки степени сжатия  (коэффициент = сжатый размер / исходный размер) и производительности (время сжатия и разжатия в секундах)
- Проведён анализ эффективности на разных типах данных
